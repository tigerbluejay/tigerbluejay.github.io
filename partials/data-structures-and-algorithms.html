Frequency Counters

The Frequency Counter is a common algorithmic pattern rather than a formal data structure. It involves counting how often values occur — typically using a hash-based structure — and then using those counts to answer questions efficiently. This pattern is especially powerful for problems involving comparisons, permutations, anagrams, or matching occurrences between datasets. Instead of repeatedly scanning collections, you precompute frequencies once and then perform constant-time lookups.

Multiple Pointers

The Multiple Pointers pattern is an algorithmic technique used to process data structures—most commonly arrays or strings—by maintaining two or more indices that move through the structure in a coordinated way. Instead of relying on nested loops to compare or combine elements, this approach allows you to make linear passes while still examining relationships between multiple positions. The key insight is that many problems involve comparing elements relative to each other rather than independently, and pointer movement can encode that logic efficiently.

Sliding Window

The Sliding Window pattern is used to process contiguous subsets of data efficiently, most often in arrays or strings. Instead of recomputing information for each possible subrange from scratch, the sliding window maintains a running state that updates incrementally as the window moves forward. This allows problems that would otherwise be quadratic to be solved in linear time.

Recursion I

Recursion is a programming technique where a function solves a problem by calling itself on smaller instances of the same problem. At a deeper level, recursion is about aligning your solution with the natural structure of the data or problem domain. Problems involving trees, graphs, nested structures, or repeated subdivision often map cleanly to recursive definitions, making recursion a powerful expressive tool rather than just a trick.

Searching Algorithms

Searching Algorithms are techniques for locating a target value within a collection of data. While the goal sounds simple, the efficiency of searching depends entirely on what assumptions you can make about the data. If the data is unsorted, you may be forced to check elements one by one. If it is sorted or structured, you can exploit that organization to eliminate large portions of the search space at once.

Linear Search Binary Search String Search

Sorting Algorithms

Sorting Algorithms arrange data into a defined order, typically ascending or descending. While sorting may seem mundane, it is one of the most deeply studied problems in computer science because it reveals fundamental limits of computation, trade-offs between time and space, and the impact of assumptions about data. Different sorting algorithms excel under different conditions, such as small datasets, nearly sorted data, or strict memory constraints.

Singly Linked List

A Singly Linked List is a linear data structure made up of nodes, where each node contains a value and a reference to the next node in the sequence. Unlike arrays, elements are not stored contiguously in memory, which means linked lists excel at insertions and deletions without requiring shifts or resizing. However, this flexibility comes at the cost of slower access, since reaching a specific position requires traversal from the head.

Structure Push / Pop Shift / Unshift Get / Set Insert / Remove Reverse / Print Rotate

Doubly Linked List

A Doubly Linked List is a linear data structure composed of nodes where each node maintains two references: one to the next node and one to the previous node. This bidirectional connectivity allows traversal in both directions and enables certain operations — especially deletions — to be performed more efficiently than in singly linked lists. Unlike arrays, linked lists do not store elements contiguously in memory, which means they avoid the cost of shifting elements when inserting or removing items.

Stacks & Queues

Stacks and Queues are linear data structures that impose strict rules on how elements are added and removed. A stack follows Last-In, First-Out (LIFO) semantics, while a queue follows First-In, First-Out (FIFO) semantics. These constraints may seem limiting, but they map naturally to many real-world and computational processes, such as function calls, undo operations, task scheduling, and buffering.

Binary Search Trees (BSTs)

A Binary Search Tree is a hierarchical data structure that organizes values so that searching, inserting, and deleting elements can be done efficiently, assuming the tree remains reasonably balanced. Each node stores a value and has up to two children, but unlike a heap, a BST enforces a global ordering rule: values smaller than a node go to the left subtree, and values larger go to the right subtree. This invariant applies recursively to every subtree, which means the entire structure represents a sorted ordering of its elements.

Binary Heap

A Binary Heap is a specialized tree-based data structure designed around one primary goal: efficiently retrieving and updating the “most important” element according to some ordering rule. Unlike many trees, a heap does not try to keep elements fully sorted. Instead, it enforces a very specific, minimal ordering constraint that is just strong enough to support fast operations like insertion and removal of the top-priority element. In a min-heap, every parent node is less than or equal to its children; in a max-heap, every parent is greater than or equal to its children. Crucially, this rule applies locally, not globally — siblings are not ordered relative to each other, and the tree is not sorted left-to-right.

Insert / Bubble Up Extract Max / Sink Down Priority Queue

Hash Tables

A Hash Table is a data structure designed to provide near-constant-time access to values based on keys. It achieves this by applying a hash function to transform a key into an index within an underlying array. Ideally, each key maps to a unique index, allowing instant retrieval. In practice, collisions occur when multiple keys hash to the same index, and the hash table must resolve them using techniques such as chaining or open addressing.

Graphs

A Graph is a highly flexible data structure used to model relationships between entities. It consists of nodes (vertices) and connections (edges), which may be directed or undirected, weighted or unweighted. Graphs are powerful because they impose very few structural constraints, allowing them to represent real-world systems such as social networks, road maps, dependency systems, and communication networks.

Dijkstra's Algorithm

Dijkstra’s Algorithm is a greedy graph algorithm designed to solve a very specific but extremely important problem: finding the shortest path from a starting node to all other nodes in a weighted graph where edge weights are non-negative. Rather than exploring all paths blindly, Dijkstra’s algorithm builds up shortest paths incrementally, always extending the currently known shortest route. The algorithm is “greedy” because at each step it commits to the locally optimal choice — the node with the smallest tentative distance — trusting that this choice will never need to be undone.

Dynamic Programming

Dynamic Programming (DP) is not a single algorithm but a problem-solving strategy for tackling complex problems by breaking them into overlapping subproblems and solving each subproblem only once. The defining insight behind DP is that many problems exhibit optimal substructure (the optimal solution depends on optimal solutions to smaller instances) and overlapping subproblems (the same computations are repeated many times). DP eliminates redundancy by storing intermediate results and reusing them.

Tries

A Trie (prefix tree) is a specialized tree structure used to store collections of strings in a way that makes prefix-based operations extremely efficient. Instead of storing complete strings as individual entries, a trie decomposes them character by character, sharing common prefixes among multiple words. This makes tries especially useful for autocomplete, spell checking, and dictionary-like applications.
